# SEO Audit

## Overview

Use this skill to identify the limiting factor before recommending changes.
Do not default to content rewrites when technical or SERP issues are the real bottleneck.

## When to Use

- Auditing a site or page for SEO issues
- Understanding why a page is not ranking or lost rankings
- Reviewing metadata, internal links, crawl/indexation, canonical tags, or sitemap issues
- Identifying the biggest SEO bottlenecks before making changes

## When Not to Use

- Use `copywriting` when the main task is writing or rewriting page copy
- Use `frontend-design` when the task is visual/UI design rather than search diagnosis

## Minimal Workflow

1. Clarify the audit scope.
2. Gather evidence from the page or site.
3. Check technical SEO basics.
4. Check relevance, information architecture, and search presentation.
5. Prioritize findings by likely impact and effort.

## Reference Routing

- Read [SCHEMA.md](SCHEMA.md) when the issue is structured data or rich-result eligibility.

## Failure Modes

- Jumping straight to content changes when crawlability or indexing is the real bottleneck
- Treating schema as separate from the broader audit when it is only one ranking or SERP factor
