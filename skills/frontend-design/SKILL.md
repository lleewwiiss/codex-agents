# Frontend Design

## Overview

Use this skill for UI creation and UI review.
It should balance visual quality, usability, and implementation realism.

## When to Use

- Building pages, components, layouts, or interaction flows
- Reviewing an existing UI for hierarchy, spacing, responsiveness, or accessibility
- Choosing a visual direction for a product surface
- Folding React or Next-specific UI advice into a concrete design decision

## When Not to Use

- Use `writing-software` for backend-only or non-visual implementation work
- Use `testing-software` for test strategy
- Do not treat every React question as a design task when the issue is purely data or module shape

## Minimal Workflow

1. Understand the interface, audience, and constraints.
2. Choose a clear visual or UX direction.
3. Keep implementation aligned with the existing design system when one exists.
4. Verify the result works on desktop and mobile.
5. Use `testing-software` for proof choice beyond visual checks and `verification-before-completion` before claiming done.

## Reference Routing

- Read [UI_REVIEW.md](UI_REVIEW.md) when auditing an existing UI.
- Read [react-rules](react-rules) for React and Next performance and rendering guidance.
- Read [remotion-rules](remotion-rules) when the task is Remotion-specific.

## Failure Modes

- Generic layouts with no intentional visual direction
- Visual flourishes that fight the existing design system
- Optimizing for novelty while making the interface harder to use
